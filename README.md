# YOLOV8-CS2

YOLOV8-CS2 是一个基于 YOLOv8 ONNX 推理与 PID 控制的 Windows 视觉自动化实验项目。项目包含屏幕区域检测、目标坐标平滑、控制参数调节、轨迹生成和外部输入接口调用等模块，适合用于学习计算机视觉实时推理、闭环控制算法和桌面自动化工程结构。

> 本项目面向算法学习、工程验证和本地实验环境。请在遵守相关法律法规、平台规则和软件服务条款的前提下使用。

## 项目概览

程序通过 ONNX Runtime 加载 YOLOv8 模型，对指定屏幕区域进行目标检测，并将检测结果交给控制模块进行平滑处理。项目提供可视化参数面板，用于调整目标保持、坐标防抖、PID 响应、轨迹平滑和切换延迟等参数，便于观察不同参数组合对实时控制效果的影响。

该仓库更偏向一个完整的实验样例，而不是通用框架。代码中保留了模型文件、Windows 调用、GUI 配置和参数保存逻辑，方便读者了解一个视觉自动化工具从检测、控制到界面封装的基本组成。

## 功能界面

打包后的程序提供 `System Host Process 配置` 窗口，核心区域为“物理引擎参数微调”面板。界面包含：

- 预设模式选择
- 远距离响应强度调节
- 近距离吸附倍率调节
- 急停防抖调节
- 模型识别防抖调节
- 静止死区调节
- 轨迹拟真度调节
- 视觉坐标平滑调节
- 目标锁定半径调节
- 目标切换延迟调节
- 垂直偏移比例调节
- 主开关控制
- 参数自动保存

这些参数主要用于本地测试和算法调试。不同硬件、模型、分辨率和运行环境下，参数效果会有明显差异。

## 核心能力

- **YOLOv8 屏幕检测**：通过 ONNX 模型对屏幕截图区域进行实时推理。
- **多后端推理支持**：优先使用 DirectML 或 CUDA，无法使用 GPU 时回退到 CPU。
- **PID 控制实验**：通过比例和微分参数观察闭环控制响应。
- **坐标平滑与防抖**：降低检测框跳动对控制输出的影响。
- **轨迹插值**：使用贝塞尔曲线生成更平滑的移动路径。
- **图形化调参**：通过 CustomTkinter 面板调整核心参数。
- **配置持久化**：参数会写入 `settings.json`，便于复现实验结果。

```text

```

## 环境要求

- Windows 10 / Windows 11
- 64 位 Python 3.9 或更高版本
- 支持 DirectML、CUDA 或 CPU 推理的运行环境
- 可正常加载本地动态链接库的 Windows 用户权限

安装依赖：

```powershell
pip install opencv-python numpy onnxruntime-directml pywin32 customtkinter
```

如果不使用 DirectML，可以改用 CPU 版本：

```powershell
pip install onnxruntime
```

## 运行方式

进入项目目录：

```powershell
cd "D:\yolov8traintool\第二次打包"
```

启动程序：

```powershell
python main.py
```

运行前请确认：

- ONNX 模型文件位于项目目录中。
- `lj.dll` 与脚本或打包后的可执行文件位于同一目录。
- Python 环境为 64 位。
- 依赖库已经安装完整。
- 如果外部接口加载失败，可尝试以管理员身份运行。

## 配置文件

`settings.json` 用于保存当前参数方案和控制参数。常见字段包括：

- `kp`：PID 比例系数
- `kd`：PID 微分系数
- `平滑`：输出移动平滑参数
- `死区`：小偏移忽略范围
- `视觉平滑`：检测坐标平滑参数
- `粘滞半径`：目标保持范围
- `切换延迟时间`：目标切换等待时间
- `锁头比例`：垂直方向目标偏移比例
- `微观磁吸`：近距离修正强度
- `坐标防抖`：坐标抖动过滤阈值

建议在修改参数前备份配置文件。参数没有通用最优值，应根据模型、屏幕分辨率、推理延迟和测试场景逐步调整。

## 注意事项

- 本项目依赖 Windows API 和外部动态链接库，不适用于 Linux 或 macOS。
- 仓库中的 ONNX 文件体积较大，克隆和拉取时需要稳定网络。
- `output/`、`__pycache__/` 和 Python 编译缓存已通过 `.gitignore` 忽略，不会作为项目源码提交。
- 如果中文文件名或配置内容显示异常，请将编辑器、终端和 Git 客户端统一为 UTF-8 编码。
- 不同版本的 ONNX Runtime、显卡驱动和 Windows 系统可能导致推理后端选择不同。
- 运行时如果出现 DLL 加载失败，请检查文件路径、系统架构和程序权限。
- 本项目没有提供通用模型训练流程，仓库中的模型仅作为当前工程的运行资产。
- 修改控制参数可能导致输出不稳定，建议先在隔离环境中验证。
